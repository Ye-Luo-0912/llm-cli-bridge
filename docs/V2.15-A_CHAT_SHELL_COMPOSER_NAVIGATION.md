# V2.15-A - Chat Shell / Composer / Navigation Consolidation Report

## ShellLayout

Baseline requested: `ad1ddd3`. Actual working baseline: current `master` at `04be1c4`, which only adds the accepted V2.14-O release readiness report on top of `ad1ddd3`.

The Bridge view now uses a modern shell:

- Left slim navigation rail: `Chat`, `Files`, `Skills`, `History`.
- No left-side Settings entry.
- Main area split into topbar, page stack, chat stream, Working Set strip, and bottom composer.
- Existing secondary panels remain available, but no longer compete with the chat stream on the default page.

## TopBar

The topbar now keeps only high-signal controls:

- Current session selector / preview.
- `+ 新聊天`.
- Settings gear that opens the existing Obsidian plugin settings modal.
- Compact runtime status: Claude Code or SDK plus connected/running/failed state.

Backend, cwd, preflight, permission, snippets, tools, and agent counters remain in code for compatibility but are no longer displayed as a permanent debug-heavy strip.

## ChatStream

The Chat page is centered on the message stream:

- User and assistant messages are rendered as chat cards.
- Command preview, workflow trace, SDK workflow, stderr, and logs remain default-collapsed.
- Failure details still expose copied/debuggable detail via existing collapsible sections.
- Run flow remains compact and folded rather than a primary debug surface.

## Composer

The composer is consolidated into a single bottom work area:

- Left tools: attachment picker, path fallback, command/preflight entry, and preset command buttons.
- Middle input: `输入消息，或使用 / 命令…`.
- Right tools: agent selector, model selector, effort/thinking selector, stop/send.
- No scattered button row below the input.

This is UI-only. Build prompt, Claude Code run, SDK run, snippets, attachments, and context toggles still use the existing handlers.

## WorkingSetStrip

The Working Set strip now sits immediately above the composer:

- Compact context chips for `Note` and `Selection`.
- FileRef chips for attachments and approved external refs.
- Empty state is a single short hint.
- The strip does not read external files or expand refs into prompt content beyond the existing bounded attachment ingestion rules.

## SecondaryPages

Files page:

- Shows Working Set / attachments / FileRef index.
- Hosts HTTP action approvals and external read requests.
- Does not become a file manager and does not execute file operations.

Skills page:

- Keeps Agent Skills before Prompt Snippets.
- Agent Skills remain runtime capabilities and do not insert into composer.
- Prompt Snippets retain explicit Insert prompt / Insert selected behavior.

History page:

- Keeps session list, preview/restore, rename, delete, sorting, and search.
- Does not carry the chat body.

## PreservedBehavior

Preserved:

- build prompt.
- Run Claude Code / SDK.
- Session/history save and restore.
- Prompt Snippets explicit insert.
- Agent Skills manifest state and inline preview.
- Working Set / native attachments.
- External read approval model.
- Existing permission panel and HTTP action approval flow.

Not added:

- Self-built write executor.
- Backup/rollback/audit transaction.
- PDF/OCR/image parser.
- SDK image streaming.
- Complex runtime tool registration.

- This is still a structural UI pass, not a pixel-perfect release polish pass.
- Icons are compact text glyphs to avoid introducing asset dependencies.
- Files page mirrors Working Set state and approvals but does not yet add richer sorting/filtering.
