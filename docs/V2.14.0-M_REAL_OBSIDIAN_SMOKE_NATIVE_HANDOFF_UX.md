## AttachmentFlow

Code-level validation confirms the existing attachment model remains intact:

- Small text / markdown / json attachments can produce bounded text snippets.
- Image, PDF, binary, and unknown files stay as FileRefs / native handoff references.
- Attachment grants remain file-scoped and do not authorize sibling files.

UX copy was tightened:

- Empty Working Set now displays a compact hint instead of disappearing.

## NativeHandoffBehavior

`promptPackage` continues to communicate the V2.14-L handoff:

- The Vault root is the working directory.
- Claude Code / SDK native file capability is responsible for ordinary Vault file execution.
- FileRef metadata indexes list authorized/user-added paths without injecting image/PDF/binary content.
- Image/PDF/binary references are handed off to Claude Code Read or SDK native capability.
