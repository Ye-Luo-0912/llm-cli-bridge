# V2.16-C — Claude-style Runtime UI / Collapsible SDK Timeline

> 基线：6ea0332（V2.16-B）。本轮在第一轮（17e3b03 SDK effective model + live progress）基础上，把 SDK-first UI 从 raw log/debug 表改成现代 Claude/Codex 风格 Agent timeline。

## BackendLabel

**目标**：SDK 运行时，消息卡片、顶部状态、timeline 均显示 SDK，不得显示 Claude Code Running/Done；fallback CLI 才显示 Claude Code fallback / Claude Code。

**实现**：
- `view.ts` 消息卡片角色标签改用 `this.actualRuntimeLabel`（动态计算），不再硬编码 "Claude Code"。
- `actualRuntimeLabel` 在 backend 探测时设置：SDK backend → `"SDK"`；CLI fallback → `"Claude Code"` / `"Claude Code fallback"`。
- 顶部状态栏 `statusLabelEl` 显示 `"SDK · 已连接"` 等，不含 "Claude Code"。
- Timeline 节点不包含 backend 名称，避免标签污染。

**Smoke 验证**：
```
✅ Backend label = SDK — label=SDK status="SDK · 已连接"
```
- `runtimeLabel === "SDK"` ✅
- 状态文本含 "SDK" ✅
- 状态文本不含 "Claude Code" ✅

## PermissionModes

**目标**：改成 Claude 风格四模式，用真实 SDK/CLI permission mode adapter 映射，popover 显示图标/标题/说明/勾选态，composer 中显示当前模式 pill。

**实现**：
- UI 暴露 4 个模式（Claude 风格）：
  | UI Label | SDK permissionMode | 说明 |
  |---|---|---|
  | Ask before edits | `default` | 每次修改需确认 |
  | Edit automatically | `acceptEdits` | 自动允许文件编辑 |
  | Plan mode | `plan` | 只规划不执行 |
  | Auto mode | `auto` | 自动允许所有操作 |
- `renderPermissionPopover()` 构建 popover：每项含图标、标题、说明、勾选态（`is-active`）。
- `togglePermissionPopover()` / `closePermissionPopover()` 控制开合。
- `setPermissionMode(mode)` 调用 adapter 映射到 SDK permissionMode，保存设置并刷新 chip。
- composer 中 `permissionModeChipEl` 显示当前模式 pill（如 "Auto mode"），点击触发 popover。
- document pointerdown + Escape keydown 关闭 popover（含 chip 点击 guard）。

**Smoke 验证**：
```

```
- popover 打开 ✅
- 4 个选项 ✅

- default 模式下 "Ask before edits" 勾选 ✅

## TimelineUI

**目标**：不再直接显示 raw LIVE 表，改成现代垂直 timeline（Thought for Ns / Agent / Bash/Read/Write/Glob/Skill / Tool output card / Coalescing / Completed）；每个节点有状态点；Tool IN/OUT 放入紧凑 code block；长路径截断 hover/title 看全路径。

**实现**：
- 新增 `timelineAdapter.ts` 纯函数模块：`adaptEventsToTimeline(events) → TimelineNode[]`。
- TimelineNode 分类：`session_started` / `thought` / `agent` / `tool_call` / `file_change` / `final_message` / `warning` / `error` / `completed` / `failed`。
- `renderTimelineNode()` 渲染单个节点：
  - 状态点（`.llm-bridge-tl-dot`）
  - session_started：标题 "Session started" + 详情（model/cwd）
  - thought：标题 "Thought" + 文本 + 持续时间
  - agent：agent 标签（Main agent / Subagent）+ 文本
  - tool_call：工具名 + 紧凑 code block（input/output）+ 持续时间
  - file_change：Created/Modified 标签 + 截断路径（title 含全路径）
  - final_message：最终回复
  - warning/error：图标 + 消息
  - completed/failed：摘要
- 长路径截断：`truncatePath(filePath, maxLen)` 超长时中间用 "..." 截断，`title` 属性含全路径。
- Tool IN/OUT 放入 `<pre>` code block，紧凑显示。

**Smoke 验证**：
```
✅ Timeline has session/tool/completed — nodes=5 kinds=session_started,session_started,tool_call,agent,completed
```
- session_started 节点存在 ✅
- tool_call 节点存在 ✅
- completed 节点存在 ✅

## AutoCollapse

**目标**：运行中默认展开当前步骤；completed 后自动折叠过程，只保留摘要；failed/error 时保留错误摘要展开；Thought、tool details、raw log 默认折叠；用户可手动展开/收起；再次运行新任务时旧任务保持折叠。

**实现**：
- `appendSdkWorkflow()` 折叠逻辑：
  - completed：timeline body 默认 `hidden`，只显示摘要（如 "Completed · 6 tools · 25s"）
  - failed：timeline body 默认展开，显示错误摘要
  - raw log：始终默认 `hidden`，独立折叠区
- 摘要格式：
  - completed：`formatCompletedSummary(stats)` → "Completed · N tools · Ns"
  - failed：`formatFailedSummary(nodes)` → "Failed · error message"
- 折叠交互：点击 toggle 切换 body hidden 状态，更新 ▶/▼ 箭头。
- 运行中：`renderLiveTimeline()` 实时渲染当前步骤，保持展开。
- 新任务时：`liveTimelineEvents` 清空，旧任务 DOM 保持不变（已折叠）。

**Smoke 验证**：
```
✅ Completed auto-collapse — autoCollapsed=true hasFailed=false
✅ Raw log collapsed — ok
```
- completed 后 timeline body 自动 hidden ✅
- raw log body 默认 hidden ✅

## EventAdapter

**目标**：新增 SDK event → UI timeline adapter；分类为 thought/agent/tool_call/tool_result/file_change/final_message/warning/error；合并重复 assistant message；recovered tool error 降级为 warning；internal runtime/private 文件写入不显示在主 timeline，只进 debug；只有用户可见 Vault 文件变更才显示 Created/Modified。

**实现**（`timelineAdapter.ts` 纯函数模块）：
- `adaptEventsToTimeline(events)` 核心逻辑：
  - `tool_start` + `tool_result` 按 `callId` 配对合并为单个 `tool_call` node（含 duration、error 标记）
  - 相邻相同文本 assistant message 合并（SDKAssistantMessage text == SDKResultMessage result 时跳过）
  - `recoverable=true` 的 error 事件降级为 `warning` node
  - `recoverable=false` 的 error 事件保持 `error` node
  - internal 路径过滤：`isInternalFilePath()` 检查 `.obsidian` / `.llm-bridge` / `.claude` / `.git` / `.env` / `node_modules` / `LLM-AgentRuntime`，internal file_change 不生成 node
- `computeTimelineStats(nodes)` 统计 tool/thought/fileChange/warning/error 计数 + 总时长
- `extractToolPath(toolName, toolInput)` 从工具输入提取文件路径
- `truncatePath(filePath, maxLen)` 长路径截断

**Smoke 验证**：
```
✅ No internal write in timeline — paths=[]
✅ No duplicate assistant — texts=1
```
- timeline 中无 internal 路径 ✅
- 无重复 assistant message ✅

## ToolSteering

**目标**：read-only 请求尽量走 Read/Glob，不应乱触发 Agent/Skill/Write；读取 manifest/name 这种任务不得创建 user-visible file；找不到文件应明确说明不存在。

**实现**：
- `promptPackage.ts` 注入 Tool Steering 指引：
  - 只读请求优先使用 Read / Glob / Grep，不要使用 Write / Edit / MultiEdit
  - 读取配置文件等只读任务，不得创建任何 user-visible 文件
  - 找不到文件时必须明确告知用户文件不存在
  - 仅在用户明确要求创建或修改文件时才使用 Write / Edit
  - 不要为读取操作创建临时文件或中间产物

**Smoke 验证**：
- Timeline 中 tool_call 节点为 Read（读取 manifest.json），无 file_change 节点（未创建文件）✅
