## ComposerPolish

The composer left side was reduced to high-frequency controls:

- Upload/native attachment picker.
- Command menu.
- Permission mode chip.

Debug and lower-frequency actions are no longer scattered as independent composer buttons. Path attachment fallback, context refresh, runtime detection, and prompt presets are now under the command menu.

The right side remains the runtime control surface:

- Agent selector.
- Model selector.
- Effort/thinking selector.
- Send/stop.

## PermissionChip

Composer now has a compact permission chip:

- `权限：受限` for `plan`.
- `权限：默认` for `default`.
- `权限：acceptEdits` for `acceptEdits`.

Clicking the chip cycles only through `plan -> default -> acceptEdits`; it saves plugin settings and refreshes status. It does not add new permission modes or runtime capabilities.
