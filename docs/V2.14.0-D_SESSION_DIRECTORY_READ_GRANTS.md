# V2.14.0-D - Session Directory Read Grants / Pending Request Queue Report

## GrantStore

V2.14.0-D adds a session-scoped pure grant store model in `src/fileAccessPolicy.ts`.

- `SessionReadGrantStore` contains `sessionReadGrants` and `pendingReadRequests`.
- `createSessionReadGrantStore()` creates an empty in-memory store shape.
- `enqueuePendingExternalReadRequest()` adds a pending read request without granting access.
- `approvePendingExternalReadRequest()` consumes a safe pending request and creates a session-scoped read grant.

No permanent external root setting is added. Grants are intended to last only for the current agent session.

## GrantRootInference

Pending external reads now propose a grant root:

- External file request defaults to the containing directory.
- External directory request proposes that directory.
- Code/project files can infer a project root from caller-provided known marker paths:
  `.git`, `package.json`, `.sln`, `.csproj`, `pyproject.toml`, `go.mod`, `Cargo.toml`.
- The module does not scan directories or read marker files. Markers must be supplied by a future caller if available.

The default approval path creates a directory-scope session grant. File-scope remains available through `forceFileScope`.

## PendingFlow

Read flow:

1. Vault-internal read remains `allow`.
2. Vault-external read without a grant returns `confirm / pending_read_request`.
3. `createPendingExternalReadRequest()` converts that evaluation into a pending request with:
   `id`, `requestedPath`, `resolvedPath`, `proposedGrantRoot`, `operation`, `risk`, `reason`, `createdAt`, `source`.
4. User approval creates a directory-scope `sessionReadGrant`.
5. Subsequent reads under that directory are allowed for the session.

Non-read operations never create pending requests.

## DirectoryGrantRules

Default session approval is directory-scoped to reduce repeated confirmations.

Attachment grants remain file-scoped. A user dragging one file into the composer authorizes that file only, not the containing directory.

Over-broad proposed roots are not automatically granted:

- Disk roots are denied.
- System directories are denied.
- User home directories require strong confirmation.
- Downloads/Desktop roots require strong confirmation.

## SafetyRules

Directory grants do not bypass sensitive path policy:

- `.env`, token, credentials, secrets, `.ssh`, private keys, `.git/config`, `.obsidian`, and `.llm-bridge` credential surfaces still deny or require strong-confirm for reads.
- Sensitive write/delete/rename remain hard denied.
- Vault-external write/delete/rename remain hard denied.
- No external file content is read.
- No external directory is scanned.
- `promptPackage`, AgentEvent v0.1, Skills runtime, CLI backend, and SDK backend are unchanged.

## Tests

The V2.14 policy unit tests now cover:

- Unapproved external read creates a pending request.
- Pending request contains the required fields.
- Project marker paths infer a project-root proposed grant.
- Approval creates a directory-scope session read grant.
- Sibling files under the approved directory are allowed.
- Sensitive sibling files remain denied.
- Write/delete/rename do not create pending requests and remain denied externally.
- External directory requests propose that directory.
- Attachment grants remain file-scoped.
- Over-broad roots are denied or require strong confirmation and are not auto-granted.

Required validation:

- `npm run build`
- `npm run test:unit`
- `npm run test:process`

## RemainingRisk

- The store is only a pure model. It is not yet wired into runtime state or UI.
- Pending approval UI still needs exact-path display, source display, risk display, and grant-root override controls.
- Real project-root inference will need a safe source of known markers; the policy module intentionally does not scan external directories.
- Directory grants reduce friction but need clear user copy because they authorize sibling files within the same session.

## Recommendation

Next phase should add the minimal in-memory runtime holder and UI surface for pending read approvals, still without reading file content or wiring prompt injection.
