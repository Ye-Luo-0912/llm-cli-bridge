## 任务 A：Blocked status glow + Thinking 重复修复

### A1. Blocked status glow

`renderAgentRunDisplayModel` header 中：
- `Needs approval` / `Needs input` → `kind="blocked"`（不含 `llm-bridge-run-glow`）
- `Running` / `Thinking` / `Reading` / `Editing` / `Running command` → `kind="running"`（含 glow）

```typescript
const isBlocked = statusText === "Needs approval" || statusText === "Needs input";
this.renderRunStatusText(head, statusText, isBlocked ? "blocked" : "running");
```

`renderRunStatusText` 仅在 `kind="running"` 时添加 `llm-bridge-run-glow` class：
```typescript
if (kind === "running") {
  statusEl.addClass("llm-bridge-run-glow");
}
```

### A2. Thinking 重复修复

`appendRunningProcessPlaceholder` 合并为单个 span，不再同时输出 `run-status-text` + `timeline-summary` 两个相同 Thinking：

```typescript
const summary = head.createEl("span", {
  cls: "llm-bridge-timeline-summary llm-bridge-run-status-text is-running llm-bridge-run-glow",
  text: "Thinking",
});
summary.setAttribute("data-run-status", "running");
```

### B2. User input 优先级守卫

当 user input 与 approval 同时 pending 时，优先显示 user input，approval panel 隐藏（pending request 不丢失，仍保留在 `pendingPermissions`）：

```typescript
// refreshPermissionPanel:
if (this.getSession().userInput.pending.size > 0) {
  panel.style.display = "none";
  this.composerBarEl?.removeClass("is-approval-active");
  return;
}
```

user input 解析后刷新 approval panel，显示之前被延迟的 approval：

```typescript

this.refreshUserInputPanel();
this.refreshPermissionPanel(); // V16.4-H: 显示延迟的 approval
```

## 任务 C：Runtime regression guard 验证

### C1. Claude SDK canUseTool

`sdkBackend.ts` canUseTool:
- `isSdkUserInputTool(toolName)` → `handleAskUserQuestion`（AskUserQuestion 专用路径，不进入 approval）
- 真实权限工具（Write/Edit/MultiEdit/Bash/Delete/NotebookEdit/WebFetch/WebSearch）→ `boundary = task.runtimePermission` → `boundary.requestApproval` / `boundary.waitForApproval`（主路径）
- 旧 `permissionState.pending` 路径仅作为 fallback（`boundary` 未注入时）

### C2. Codex app-server approval

`codexAppServerApprovalMapper.ts`:
- `commandExecution` → toolName="Bash", riskLevel="high"
- `fileChange` → toolName="Write", riskLevel by fileAction (delete=high, create/modify=medium)
- `declineForSession` → codex 协议层 "decline"（codex 无 declineForSession）；会话级 deny 缓存由 PermissionBoundary 内部 `deniesList` 维护

### C3. AskUserQuestion

`isSdkUserInputTool` 识别 `askuserquestion` 及其他 user input tool names，走 `handleAskUserQuestion` → UserInputBoundary，不进入 approval card。

### C4. Permission mode

`bridgeSession.ts` `rebuildPermissionBoundary`:
- `currentRunId !== null` 时不重建（run 进行中不改变当前 run 的 boundary）
- `permission.mode === settings.claudePermissionMode` 时不重建（mode 未变）
- 重建后 `this.permission` 指向新 boundary，下一次 run 使用新 mode
