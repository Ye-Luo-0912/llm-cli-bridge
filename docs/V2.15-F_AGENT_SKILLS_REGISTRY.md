# V2.15-F Agent Skills Registry / Native Detail / Composer Polish

## AgentSkillsRegistry

Skills page now acts only as an Agent Skills registry. It lists runtime capabilities from `.llm-bridge/agent-skills.json` and shows compact skill cards with name, slug, source, description, and enabled state.

Prompt Snippets are not restored. The page does not render legacy snippet import, insert, append, combo, search, group, sort, or preview controls.

## ObsidianNativeDetail

Clicking an Agent Skill opens its materialized `.claude/skills/<slug>/SKILL.md` in an Obsidian leaf. The plugin resolves absolute materialized paths back to vault-relative paths when possible.

If the file cannot be opened through the Vault index, the plugin shows a clear Notice and copies the path as a fallback. Instructions are never copied to the composer.

## NoPromptSnippetsRegression

Agent Skills remain runtime capabilities. They are not prompt snippets, are not inserted into the input textarea, and are not concatenated into `promptPackage`.

Enable/disable only updates the Agent Skills manifest. Refresh only reloads manifest/materialized state.

## ComposerFullInput

The composer now gives the textarea the full main visual area. Upload, command, and permission controls sit in the lower-left overlay. Model/effort and send/stop controls sit in the lower-right overlay.

Textarea bottom padding is increased so text is not hidden behind controls. Clicking composer whitespace focuses the textarea.

## ModelEffortPicker

The native model-by-effort select was replaced with a custom compact picker. The chip displays `model · effort`, for example `glm-5.2 · 高`.

The popover separates model choices from effort choices. Effort labels are Chinese: 低 / 中 / 高 / 超高. Selections still write `settings.model` and `settings.effortLevel` separately.

## RemainingRisk

Obsidian does not index hidden `.claude` files as `TFile` in the tested Vault. The plugin can verify the file through the adapter and can provide a fallback Notice, copy the absolute path, and locate it in the system file manager, but Obsidian native Markdown opening requires the file to be indexed by the Vault.
