# V2.14.0-G - Composer Attachments / Working Set / Lightweight Ingestion Report

## AttachmentFlow

Composer now has a minimal `+ File` entry for user-provided attachment paths.

- User-provided paths create `attachment` FileRefs.
- Attachment refs create `attachmentReadGrant` records with `match=file`.
- Attachment refs enter the in-memory Working Set.
- Attachment grants do not authorize sibling files or the containing directory.

## WorkingSetUI

The composer renders compact Working Set chips.

- Chips show `displayName`, `kind`, `status`, `source`, and `pathKind`.
- Bounded text ingestion is marked on the chip.
- Removing a ref removes it from the Working Set.
- Removing an attachment ref also removes its attachment read grant and any bounded snippet.

## TypeClassification

File type classification is extension-based only.

- `markdown`
- `json`
- `text`
- `image`
- `pdf`
- `binary`
- `unknown`

No deep file probing, OCR, PDF parsing, image parsing, or directory scanning is added.

## BoundedIngestion

Bounded ingestion applies only to user-added attachment refs.

- Only `text`, `markdown`, and `json` are eligible.
- Files above `MAX_ATTACHMENT_INGEST_BYTES` remain FileRefs only.
- Text above `MAX_ATTACHMENT_INGEST_CHARS` is truncated with a clear marker.
- Sensitive paths are not read.
- Image, PDF, binary, and unknown files are kept as FileRefs only.

## Tests

Added coverage verifies:

- Attachment ref creation.
- File-scope attachment grant creation.
- Working Set chip/list rendering hooks.
- Ref removal and attachment grant cleanup hooks.
- Type classification for markdown/json/text/image/pdf/binary.
- Bounded ingestion for small text attachments.
- Oversized files are not read.
- Image/PDF/binary files are not read.
- Sensitive attachments are not read.
- External sibling remains unauthorized.
- `promptPackage` does not include unauthorized external refs.

Required validation:

- `npm run build`
- `npm run test:unit`
- `npm run test:process`

## RemainingRisk

- Drag-and-drop is not implemented; this phase uses an explicit path-entry button.
- Attachment UX is intentionally compact and not a full file manager.
- No OCR, PDF parsing, image embedding, or base64 prompt injection is included.
- Real file tool callers are still not connected.

## Recommendation

Freeze G after validation. The next phase can add drag/drop or a native file picker, then separately add file-tool callers behind the existing read/write policy boundary.
