# V2.14.0-K1 - Runtime File Tool Adapter Limits Hardening Report

## LimitClamp

Runtime-provided limits are now treated as upper-bound requests only.

- `maxReadBytes` is clamped to `DEFAULT_FILE_TOOL_MAX_READ_BYTES`.
- `maxReadChars` is clamped to `DEFAULT_FILE_TOOL_MAX_READ_CHARS`.
- `maxListEntries` is clamped to `DEFAULT_FILE_TOOL_MAX_LIST_ENTRIES`.
- `maxListDepth` is clamped to `DEFAULT_FILE_TOOL_MAX_LIST_DEPTH`.
- `maxSearchFiles` is clamped to `DEFAULT_FILE_TOOL_MAX_SEARCH_FILES`.
- `maxSearchResults` is clamped to `DEFAULT_FILE_TOOL_MAX_SEARCH_RESULTS`.
- `maxSearchBytesPerFile` is clamped to `DEFAULT_FILE_TOOL_SEARCH_BYTES_PER_FILE`.

Runtime input can still pass smaller values for more conservative execution. Invalid, non-integer, zero, or negative limits are ignored instead of expanding scope.

## SearchExtensionPolicy

Search extensions are now a subset of the executor default allowlist.

- The executor exports `DEFAULT_FILE_TOOL_SEARCH_EXTENSIONS`.
- Runtime adapter normalizes extensions to lowercase dot-prefixed values.
- Unsupported extensions such as `.pdf`, `.png`, `.bin`, and `.exe` are filtered out.
- Runtime input can narrow the search extension set, but cannot expand it.

## NoWriteBoundary

No write capability is added.

- Runtime file tools remain read-only: `stat`, `read`, `list`, `search`.
- `write`, `delete`, `rename`, and diff-write remain unsupported.
- Confirm pending flow, I1 realpath/symlink guard, refs-only media behavior, and bounded read behavior remain unchanged.

## Tests

Added coverage verifies:

- Oversized runtime `maxReadBytes` and `maxReadChars` are clamped.
- Oversized list/search limits are clamped.
- Smaller runtime limits are preserved.
- Search extensions can only narrow the executor default allowlist.
- Invalid limits are ignored.
- `write`/`delete`/`rename` remain unsupported.

Validation:

- `npm run build`
- `npm run test:unit`
- `npm run test:process`

## Recommendation

Freeze K1 if build/unit/process pass and diffs remain limited to adapter clamp logic, executor constant export, tests, and report. Future runtime tool adapters must use these clamped limits and must not pass raw tool input limits directly to the executor.
