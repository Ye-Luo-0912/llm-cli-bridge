## 1. RootCause（根因分析）

**Blocker**: 导入 Skill 重命名后 `pinned/applyCount/lastUsedAt/groupOverride` 未迁移到新名称。

**根因**: `view.ts` 中 `openEditSkillDialog` 的时序冲突：

1. `renameSkillMeta(state, oldName, newName)` — 内存中迁移 meta ✅
2. `scheduleSkillsStateSave()` — 500ms 防抖定时器，**不立即写盘** ❌
3. `refreshSkills()` — 立即从磁盘重载 `skillsState`，**覆盖内存迁移** ❌

时序图（V2.11.1 旧逻辑，有 bug）：
```
renameSkillMeta → scheduleSkillsStateSave(500ms) → refreshSkills(立即)
                      ↓                              ↓
                 定时器待触发                    loadSkillsState 从磁盘读旧 state
                                                   ↓
                                             内存迁移被覆盖，旧名 meta 残留
```

**关键**: `renameSkillMeta` 函数本身正确（skillsState.ts:215-224），问题在调用时序。`refreshSkills` 在防抖定时器触发前就从磁盘重载了旧 state，导致内存迁移被覆盖。
