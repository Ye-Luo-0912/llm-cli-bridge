# 测试约定

项目进入收口阶段后，不再维护集中式全量测试文件。

- 每次修改只补与本次风险直接相关的独立 smoke；一个脚本只验证一个能力或一条用户流程。
- 默认验证为 `npm run build`；涉及 Runtime 协议时运行对应的真实协议 smoke，涉及 UI 时运行对应的 UI smoke。
- 测试不得默认读写真实 Vault；需要真实 Obsidian 的验证必须显式指定目标并由人工发起。
- 禁止用源码字符串断言实现细节，禁止为了旧测试修改正确的产品行为。
- 临时测试完成且不再承担发布门禁时直接删除，不累计历史测试段。
- 历史测试报告只作版本记录，不代表当前发布流程仍需执行对应命令。

发布前按本次改动选择最小验证集，不恢复统一的 `test` / `test:unit` 入口。
