## 1. RootCause（根因分析）

**Blocker**: Bridge View 顶部控制区、Skills/History 区域存在严重拥挤与错位，按钮漂浮到下一行或遮挡标题。

**根因（CSS 遗漏 + 布局约束缺失）**:

1. **5 个 class 完全未在 styles.css 中定义**（V2.3/V2.5/V2.8 引入 DOM 但遗漏 CSS）：
   - `.llm-bridge-sb-advanced-toggle`（V2.4 Advanced 折叠按钮）
   - `.llm-bridge-skills-import-btn`（V2.3 Skills 导入按钮）
   - `.llm-bridge-history-head`（V2.5 History 面板 header）
   - `.llm-bridge-history-toggle`（V2.5 History 折叠 toggle）
   - `.llm-bridge-history-refresh-btn`（V2.5 History 刷新按钮）

2. **History 面板整体 CSS 缺失**：V2.5 引入 history-panel/history-body/history-search/history-list/history-item/history-title/history-meta 等十余个 class，但 styles.css 中完全未定义，导致 History 列表无布局约束、按钮无样式。

3. **折叠区 head 未统一 flex 布局**：`.llm-bridge-skills-head`/`.llm-bridge-run-flow-head` 仅设 padding/cursor，未设 `display:flex; align-items:center;`，导致 header 内按钮（导入/刷新）漂浮到下一行或与标题错位。`.llm-bridge-history-head` 完全未定义。

4. **输入框缺 overflow-y**：`.llm-bridge-input` 有 `max-height:160px` + `resize:none` 但缺 `overflow-y:auto`，超长文本溢出。

5. **状态栏无 flex-wrap**：`.llm-bridge-status-bar` 子元素多时无换行约束，导致顶部拥挤。
