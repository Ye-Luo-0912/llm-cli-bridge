# V2.16-D User-facing Agent UX / Context API / File Input

## Scope

- Baseline: current `master` at implementation time.
- No release/tag.
- No Prompt Snippets restoration.
- No new runtime capability, write executor, PDF/OCR/image parser, SDK image streaming, or complex runtime tool registration.
- Goal: move the SDK-first UI away from developer/debug noise toward a user-facing coding-agent workspace.

## Session Dropdown

- Recent sessions now include:
  - title
  - first user request summary
  - last assistant reply summary
  - saved time
  - message count
- Current session is highlighted in the dropdown.
- Clicking a recent session restores that session directly instead of navigating to History.
- `查看全部历史` remains the explicit navigation entry for the full History page.
- Restored session state includes messages, working set refs, attachment file grants, model, effort, permission mode, backend mode, and session mode.
- `keepLastSession` continues to restore the last active session on view open when enabled.

## Context API / Usage Source

- The context indicator no longer presents local estimated tokens as authoritative usage.
- Until SDK/CLI exact usage APIs are wired, the primary state is `unavailable`.
- Local estimation is kept only in details and labelled as estimated.
- The context detail view includes prompt, active note, selection, working set, history, remaining, and compression information when present.
- Compression display supports before/after/ratio/source/reason, but no new runtime compression behavior is added here.

## File Input

- Composer supports:
  - direct drag/drop file input
  - clipboard file paste when Electron exposes file paths
  - pasted path text recognition
  - `@` Vault file picker from the existing inline mention flow
- Vault files are added as `vault` FileRefs.
- User-provided external files are added as attachment FileRefs with file-scope attachment read grants.
- Images/PDF/binary/unknown files remain native refs only.
- Small text/markdown/json attachments keep the existing bounded ingestion behavior.
- Sensitive paths are not ingested.
- If a native file picker or paste event does not expose a path, the UI reports the reason and does not treat the action as success.

## User Mode / Developer Mode

- `developerMode` is a setting and defaults to `false`.
- Default user mode hides raw command preview, workflow trace, debug log, raw SDK log, and internal runtime/private file timeline entries.
- Developer mode exposes those raw details for diagnosis.
- Failed messages still show a user-facing error summary, with raw stack/log gated behind Developer mode.

## Agent Timeline

- Running timeline keeps user-facing steps.
- Completed/stopped workflows in user mode remove live/process details and show only the final assistant output.
- User mode filters low-level `session_started`, `thought`, and internal/private file events.
- Developer mode keeps workflow/timeline/raw details available for debugging.

## Topbar / Composer

- Topbar retains the compact Bridge/session/new/settings/runtime status layout.
- Long titles rely on existing truncation behavior.
- Context information stays in the compact context strip instead of expanding the topbar.
- Composer remains a full-area textarea with drag/drop and paste file handling.

## Remaining Risk

- Exact SDK/CLI token usage is still unavailable in the current plugin boundary. The UI now states that clearly instead of promoting local estimates.
- Clipboard file path support depends on Electron/Obsidian exposing `File.path`; when unavailable, users must use the Vault picker or path fallback.
- External user attachments are read-only grants for the specific file, not directory grants.

## Recommendation

Freeze V2.16-D after build/unit/process verification. Next work should wire exact runtime usage only if SDK/CLI exposes stable usage APIs; otherwise keep `unavailable` as the honest user-facing state.
