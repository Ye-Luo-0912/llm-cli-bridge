# V2.15-E Remove Prompt Snippets / Restore Agent Skills Semantics

## RemovalDecision

V2.15-E removes Prompt Snippets from the release-candidate UI and main runtime flow.

Skills now means Agent Skills only: runtime capabilities discovered or invoked by Claude Code / SDK. Agent Skills are managed through `.llm-bridge/agent-skills.json` and materialized under `.claude/skills/<slug>/SKILL.md`; they are never inserted into the composer and are not concatenated into `promptPackage`.

## RemovedPromptSnippets

- Removed the Skills page Prompt Snippets section.
- Removed legacy import, edit, delete, seed, search, group, sort, pin, combo, Insert prompt, Append, and Insert selected UI from `src/view.ts`.
- Removed composer command-menu preset buttons to avoid keeping a prompt-template insertion surface beside Agent Skills.
- Removed snippets status-bar counting and skills-state lifecycle flushes from the Bridge view.
- Removed Prompt Snippet modal classes from the Bridge view.
- Removed dead preset/snippet control styles from `styles.css`.

## AgentSkillsOnlyPage

- Skills page renders only the Agent Skills panel.
- Agent Skills show enabled/disabled state, slug/source/materialized path, updated metadata, and instructions preview.
- Clicking an Agent Skill selects it and updates the inline preview only.
- Enabled toggles update `.llm-bridge/agent-skills.json`.
- No Agent Skill click path writes to `inputEl` or prompt composer state.

## LegacyDataHandling

Legacy `.llm-bridge/skills.md` and `.llm-bridge/skills/` data is not auto-deleted to avoid destructive upgrade behavior, but V2.15 no longer reads, displays, imports, edits, applies, or writes that data from the Bridge UI.

The old parser modules may remain temporarily as dormant code for compatibility cleanup, but they have no Bridge UI entry, no runtime call path, and no composer insertion path.

## DocsUpdate

- `README.md` now describes Skills as Agent Skills runtime capabilities only.
- `docs/USER_GUIDE.md` removes Prompt Snippets steps and documents legacy data as unused.
- `RELEASE_CHECKLIST.md` adds release checks for Agent Skills-only UI and absence of Insert prompt / Insert selected / Append.
