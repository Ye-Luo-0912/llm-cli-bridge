# V2.15-E RC UI Regression Fix / Attachment & Skills Repair

## AttachmentRepair

- Composer `+` now opens an attachment menu instead of assuming the native picker can provide `File.path`.
- Menu entries: 添加 Vault 文件, 添加外部路径, 从剪贴板路径添加, 原生文件选择器.
- Vault file selection uses `app.vault.getFiles()` and adds the selected file as a user attachment ref.
- Native picker failures without a real path show a fallback notice and do not count as success.

## SkillsRegressionFix

- Agent Skills remain runtime capabilities and only preview/toggle state in the UI.
- Prompt Snippet row clicks now preview only.
- Composer insertion requires explicit `Insert prompt`, `+`, or `Insert selected`.
- Prompt Snippets and Agent Skills remain separate panels.
- Follow-up correction: the Skills page no longer mounts the legacy Prompt Snippets panel at all. Legacy snippet insertion code remains hidden/legacy, but the Skills workflow is Agent Skills only and cannot insert prompt text into composer.

## SessionDropdown

- The current session selector opens a recent sessions dropdown.
- Recent sessions can be restored directly from the dropdown.
- `查看全部历史` is the only dropdown action that switches to the History page.

## ComposerInputArea

- The composer no longer shows a persistent Claude Code agent selector.
- The textarea occupies the primary visible composer area.
- Clicking blank composer space focuses the textarea.
- Left controls stay limited to attachment, command, and permission mode.
- Right controls stay limited to model/effort and send/stop.

## ModelEffortDropdown

- Model and thinking effort are merged into a compact dropdown such as `glm-5.2 · High`.
- Runtime/backend selection remains outside the always-visible composer controls.

## IconPolish

- The Chat rail icon now uses `message-square`, not a phone symbol.
- Rail stays icon-only for Chat, Files, Skills, and History.

## DetailsCollapse

- Run Flow remains collapsed by default when runs start or finish.
- Command/workflow/log technical details continue to require explicit expansion.
