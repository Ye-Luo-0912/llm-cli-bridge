# V16.4-G Agent Status & Approval UX

## 概述

V16.4-G 在 V16.4-F2（PermissionBoundary session cache + run permission snapshot fix）基础上，对 approval 语义与 running status UI 做 polish：

- **任务 A**：拆分 `decline`（deny once）与 `declineForSession`（deny session），让 UI 与 PermissionBoundary 行为对齐。
- **任务 B**：running 状态从旋转圈改为 Codex-style glow text，blocked 状态（Needs approval / Needs input）不用 shimmer。
- **任务 C**：清理旧 spinner / 旧 perm-card CSS，新增 `.llm-bridge-run-status-text` 系列样式 + `prefers-reduced-motion` 降级。
- **任务 D**：本文档 + 行为测试 + 测试报告。

## 任务 A：Approval decline 语义修正

### ApprovalResponse 扩展

`src/runtime/core/types.ts`：

```typescript
export type ApprovalResponse =
  | { type: "accept" }            // 允许本次（allow once）
  | { type: "acceptForSession" }  // 本会话允许（allow session）
  | { type: "decline" }           // 只拒绝本次，不写 deniesList（deny once）
  | { type: "declineForSession" } // 本会话拒绝，写 deniesList（deny session）
  | { type: "cancel" };           // 取消整个 run
```

### PermissionBoundaryImpl.resolveApproval 行为

`src/runtime/core/permissionBoundary.ts`：

- `acceptForSession` → 写 `allowsList`（pathPattern = mergeKey）
- `decline` → **不写** `deniesList`（只拒绝本次）
- `declineForSession` → 写 `deniesList`（pathPattern = mergeKey）
- `accept` / `cancel` → 不写任何缓存

`requestApproval` 先按 mergeKey 匹配 allowsList/deniesList（V16.4-F2 修复），所以：

- `decline` 后同 mergeKey 第二次仍 `pending`
- `declineForSession` 后同 mergeKey 第二次 `auto-deny`
- `acceptForSession` 后同 mergeKey 第二次 `auto-allow`

### PermissionChoice 扩展

`src/sdkPermission.ts`：

```typescript
export type PermissionChoice = "allow_once" | "allow_session" | "deny_once" | "deny_session";
```

### UI approval card 按钮

`src/view.ts` refreshPermissionPanel：

| 按钮文案 | CSS class | PermissionChoice | ApprovalResponse |
|---|---|---|---|
| Yes, proceed | `is-proceed` | `allow_once` | `accept` |
| Yes, don't ask again for this session | `is-proceed-session` | `allow_session` | `acceptForSession` |
| No, skip this once | `is-decline` | `deny_once` | `decline` |
| No, don't ask again this session | `is-decline-session` | `deny_session` | `declineForSession` |

### Codex / SDK mapper 一致性

- `codexAppServerApprovalMapper.ts`：`declineForSession` → `"decline"`（codex server 协议无 declineForSession，会话级 deny 由 PermissionBoundary 维护）
- `sdkBackend.ts` canUseTool：`decline` / `declineForSession` / `cancel` 都返回 `deny`；`declineForSession` 的会话级 deny 缓存由 PermissionBoundary 内部维护
- `assistantTurnView.ts` approvalResponseToLegacyChoice：`decline` → `deny_once`，`declineForSession` → `deny_session`
- `providerLifecycleEvent.ts` approvalResolution 类型增加 `declineForSession`

## 任务 B：Running status UI 改成 Codex-style glow text

### renderRunStatusText 方法

`src/view.ts` 新增统一方法：

```typescript
private renderRunStatusText(
  parent: HTMLElement,
  text: string,
  kind: "running" | "blocked" | "completed",
): void
```

- `kind="running"`：shimmer/glow text（Thinking / Reading <file> / Running command / Editing <file>）
- `kind="blocked"`：无 shimmer（Needs approval / Needs input）
- `kind="completed"`：无 shimmer（静态文本）

### Spinner 替换

所有 `.llm-bridge-msg-spinner` / `.llm-bridge-turn-header-spinner` 使用点替换为 `renderRunStatusText`：

1. `renderMessage` msg-head running 态（line ~3187）
2. `appendRunningProcessPlaceholder`（line ~3454）
3. `renderAgentRunDisplayModel` turn header（line ~3508）
4. `updateMessageBlock` 动态更新（line ~4934）
5. `renderAgentRunDisplayModel` currentActivity（line ~3540）— blocked 状态用 `is-blocked` kind

### phase card 降噪

- running phase 用 glow text 或 subtle pill
- completed phase 默认不显示 completed tag（由 phaseModel.resultSummary 提供语义化摘要）

### prefers-reduced-motion 降级

```css
@media (prefers-reduced-motion: reduce) {
  .llm-bridge-run-status-text.is-running.llm-bridge-run-glow {
    background: none;
    -webkit-text-fill-color: var(--interactive-accent);
    animation: none;
  }
  .llm-bridge-status-dot-running {
    animation: none;
  }
}
```
