# V2.14.0-I1 - File Tool Executor Realpath / Symlink Hardening Report

## RealpathCheck

`fileToolExecutor` now resolves the actual execution target before touching file content or enumerating directories.

- Initial request still goes through `evaluateFileToolPolicy`.
- After initial allow, executor uses `lstat` and `realpath`.
- If realpath differs from the requested/resolved path, executor re-runs `evaluateFileToolPolicy` against the real target.
- `confirm` returns a pending request and does not execute.
- `deny` returns a refusal and does not execute.

## SymlinkPolicy

Symlink behavior is hardened without adding new file capabilities.

- Vault-local symlink to unauthorized external file is not directly allowed.
- Symlink target inside an authorized session root may be allowed after realpath re-check.
- Requested path and realpath target both go through sensitive-path checks.
- Sensitive realpath target is denied.
- No write/delete/rename execution is added.

## ListSearchTraversalSafety

Directory traversal avoids symlink escape.

- `list` uses `lstat` for child entries and skips symlinks.
- `search` uses `lstat` and skips symlink files/directories.
- Search does not recurse through symlink directories.
- List does not expose metadata for symlink targets outside the authorized root.

## Tests

Added coverage verifies:

- Vault symlink to external ordinary file returns confirm for read/stat, not allow.
- Vault symlink to external sensitive file returns deny.
- Authorized directory containing symlink to external directory does not list or search target contents.
- Ordinary authorized files still read/list/search normally.
- External session grant allows a symlink only when the realpath target is inside the grant root.
- Static guard verifies `lstat`, `realpath`, `isSymbolicLink`, and policy re-check remain in executor.

Required validation:

- `npm run build`
- `npm run test:unit`
- `npm run test:process`

## Recommendation

Freeze I1 if build/unit/process pass and diffs remain limited to executor, tests, and report. Future file tool routing should rely on this executor instead of reading paths directly.
