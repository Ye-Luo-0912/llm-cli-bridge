### AbortSignal 兼容性修复

**根因**: SDK 源码 `import{setMaxListeners as Wz}from"events"` 是 ESM import，解析到 Node 内置 events 模块。ESM 模块缓存与 CJS `require.cache` 完全独立，CJS 层 patch `events.setMaxListeners` 无效。SDK 调用 `Wz(e, t.signal)` 传入 browser AbortSignal，Node 的 setMaxListeners 检查 `instanceof EventEmitter` 失败。

**解决方案**: `installNodeCompatibleAbortController()` — 在 SDK query 调用前，将 `globalThis.AbortController` 替换为 EventEmitter-based 版本。SDK 内部 `new AbortController()` 得到 EventEmitter-based signal，通过 `instanceof EventEmitter` 检查。query 结束后恢复原始 AbortController。

**文件**: `src/sdkBackend.ts` 行 89-142, 行 727-728 (安装), 行 821-822 (恢复)
