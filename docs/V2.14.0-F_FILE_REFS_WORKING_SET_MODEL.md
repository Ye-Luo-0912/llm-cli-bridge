# V2.14.0-F - File Refs / Working Set Model Report

## FileRefModel

V2.14.0-F adds a shared `FileRef` model for file context references.

Required fields:

- `id`
- `kind`
- `displayName`
- `requestedPath`
- `resolvedPath`
- `pathKind`
- `source`
- `grantScope`
- `createdAt`
- `status`

Supported `kind` values:

- `vault`
- `external`
- `attachment`

The model stores file references only. It does not include file body, extracted text, image data, PDF text, or parsed content.

## WorkingSetRules

`WorkingSet` contains only `FileRef[]`.

- Vault refs can be added when the read policy already allows the path.
- External refs are not added while authorization is pending.
- External refs are added only after a session read grant is created.
- Attachment refs represent user-selected files and are active immediately.
- Session/view cleanup clears the in-memory working set and attachment grants.

## GrantIntegration

Grant behavior:

- Attachment refs create an `attachmentReadGrant` with `match=file`.
- External unauthorized read requests create pending read requests through `queueExternalFileAccessRequest()`.
- `queueExternalFileAccessRequest()` now accepts `pathKind`, `knownProjectRootMarkers`, and `source`.
- Approved external read requests can be converted into active external refs backed by session read grants.
- Directory grants and file grants remain authorization metadata only.

## PromptBoundary

This phase does not connect file refs to prompt construction.

- No external file content is read.
- No external directory is scanned.
- No image/PDF parsing is added.
- `promptPackage` is not modified and does not receive file ref text.
- CLI backend, SDK backend, AgentEvent v0.1, and Skills runtime remain unchanged.

## Tests

Added coverage verifies:

- Vault refs can enter the working set.
- Unauthorized external reads generate pending requests and do not enter the working set.
- Approved external session grants allow external refs to enter the working set.
- Attachment refs automatically create file-scope attachment grants.
- Working set refs do not contain file body fields.
- `promptPackage` remains unwired.
- View runtime exposes in-memory working set methods without reading files.

Required validation:

- `npm run build`
- `npm run test:unit`
- `npm run test:process`

## RemainingRisk

- No real composer drag/drop or attachment picker is connected yet.
- No file tool caller is connected yet.
- No UI list for Working Set refs is rendered in this phase.
- Project-root inference still depends on caller-provided `knownProjectRootMarkers`; the plugin does not scan external directories to discover markers.

## Recommendation

Freeze F after validation. The next phase can add a minimal Working Set UI or a narrow runtime caller that creates file refs, while keeping file content reads and prompt injection out of scope.
