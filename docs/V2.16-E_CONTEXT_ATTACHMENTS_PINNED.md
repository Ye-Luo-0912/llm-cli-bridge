# V2.16-E - Context / Attachments / Pinned Context

## DesignShift

Working Set is no longer a required primary UI concept.

The user-facing model is now:

- Attachments: files attached to the current message.
- Context: lightweight Note / Selection toggles for the current run.
- Pinned context: explicit cross-turn refs selected by the user.

Vault/cwd remains the agent workspace. Claude Code / Claude SDK should explore Vault files with native tools when needed.

## AttachmentScope

- Drag/drop, paste, `@` selection, and path input default to `scope=message`.
- Message-scoped attachments appear inside the composer and then move to the corresponding user message after send.
- Composer attachments are cleared immediately after send.
- Message-scoped attachments are not included in subsequent prompts.
- Clicking `Pin` creates `scope=pinned`, which persists across turns until removed.

## PackingPolicy

`AttachmentPackingPolicy` defines:

- `smallTextInlineMaxBytes`
- `smallTextInlineMaxChars`
- `allowedTextExtensions`
- `binaryAsNativeRef`
- `imageAsSdkAttachmentIfSupported`
- `sdkDirectAttachmentSupported`

Small text / markdown / json files are bounded inline for the current run. Large files, PDF, binary, and unknown files stay as path refs unless SDK support is explicitly tested.

## SdkStreamingInput

Official Claude Agent SDK documentation states `query()` accepts `prompt` as either a string or an `AsyncIterable<SDKUserMessage>`. The latter is Streaming Input.

Implementation:

- No attachments: keep string prompt.
- Small text attachments: inline in promptPackage for the current run.
- Image / screenshot / blob attachments: cached locally, then converted to SDK image content blocks when building `sdkStreamingInput`.
- PDF/document blocks are not enabled by default because local package/types were not installed in this workspace for direct verification.
- Claude Files API upload is not used by default because it is remote API-level file storage and requires explicit user opt-in.

## PromptBoundary

PromptPackage now separates:

- Message attachment refs.
- Pinned context refs.
- Session authorization refs.
- Message Attachments Inline bounded snippets.

Pinned refs are the only file refs that persist into later runs automatically.

## Tests

Automated coverage was added for:

- FileRef scope defaults.
- Composer message attachments moving to user message.
- Pinned context persistence.
- AttachmentPackingPolicy fields.
- PromptPackage message/pinned/session sections.
- SDK Streaming Input path for image blocks.
- No always-visible empty Working Set UI.

## RemainingRisk

The local workspace did not contain `node_modules/@anthropic-ai/claude-agent-sdk`, so SDK direct attachment type validation is based on official documentation and runtime-compatible object shape. Image blocks use the documented Streaming Input shape; PDF/document content blocks remain disabled until local SDK package validation confirms stability.

## Recommendation

Freeze this direction: keep message attachments ephemeral by default, inline only bounded small text for efficiency, use SDK Streaming Input for images when SDK is active, and require explicit Pin for cross-turn context.
