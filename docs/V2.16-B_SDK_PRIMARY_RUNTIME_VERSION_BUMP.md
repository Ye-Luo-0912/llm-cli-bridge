## RuntimeStrategy

### BackendMode 类型变更

| 旧值 | 新值 | 说明 |
|------|------|------|
| auto | auto | 含义变更：SDK-first + CLI fallback（原为直接走 CLI） |
| - | cli | 新增：始终使用 ClaudeCliBackend（显式选择） |
| sdk-experimental | sdk | 重命名：始终使用 SdkBackend（显式选择，不可用时不静默 fallback） |
| mock-success | mock-success | 不变 |
| mock-failure | mock-failure | 不变 |

### getBackend 策略 (view.ts)

```
auto → isSdkAvailable(vaultPath) ? SdkBackend(strict=false) : ClaudeCliBackend
cli  → ClaudeCliBackend（始终）
sdk  → SdkBackend(strict=true)（SDK 不可用时发 failed 事件，不 fallback mock）
mock-* → MockAgentBackend
```

### SdkBackend strict 模式

- `strict=false`（auto 模式）：SDK 不可用时 fallback mock workflow（兼容旧行为）
- `strict=true`（显式 sdk 模式）：SDK 不可用时发 `error` + `failed` 事件，显示明确错误消息，不静默 fallback

### Settings 迁移

- `main.ts` loadSettings：检测到 `"sdk-experimental"` 自动迁移为 `"sdk"` 并保存
- 不强制覆盖已有用户设置（仅迁移已废弃的枚举值）

## RuntimeStatusUI

### 顶部 runtime status 显示实际 backend

`refreshStatusBar()` 根据 backendMode 和 SDK 可用性显示：

| backendMode | SDK 可用 | 显示标签 |
|-------------|----------|----------|
| auto | 是 | `SDK · 已连接` |
| auto | 否 | `Claude Code fallback · 已连接` |
| cli | - | `Claude Code · 已连接` |
| sdk | 是 | `SDK · 已连接` |
| sdk | 否 | `SDK unavailable · 已连接` |
| mock-* | - | `Mock · 已连接` |

### model picker 不负责切 backend

- model picker 仅控制 `settings.model` 和 `settings.effortLevel`
- backend 切换通过设置页 Backend 模式下拉
